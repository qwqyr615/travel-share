# 数据库设计

## user - 用户表

| 字段         | 类型         | 约束                       | 说明               |
| ------------ | ------------ | -------------------------- | ------------------ |
| id           | INT          | PRIMARY KEY AUTO_INCREMENT | 用户ID             |
| username     | VARCHAR(50)  | NOT NULL, UNIQUE           | 用户名（登录用）   |
| password     | VARCHAR(100) | NOT NULL                   | BCrypt 加密存储    |
| nickname     | VARCHAR(50)  | DEFAULT NULL               | 昵称               |
| avatar       | VARCHAR(255) | DEFAULT NULL               | 头像图片路径       |
| intro        | VARCHAR(300) | DEFAULT NULL               | 个人简介           |
| type         | Byte         | NOT NULL DEFAULT 'user'    | 角色：user / admin |
| created_time | DATETIME     | DEFAULT CURRENT_TIMESTAMP  | 注册时间           |

```mysql
CREATE DATABASE IF NOT EXISTS travel
    DEFAULT CHARACTER SET utf8mb4
    DEFAULT COLLATE utf8mb4_0900_ai_ci;

USE travel;

DROP TABLE IF EXISTS t_user;

CREATE TABLE t_user (
                        id          INT PRIMARY KEY AUTO_INCREMENT,
                        username    VARCHAR(50)  NOT NULL,
                        password    VARCHAR(100) NOT NULL,
                        nickname        VARCHAR(50),
                        avatar     VARCHAR(255),
                        intro       VARCHAR(255),
                        -- 0: USER, 1: ADMIN
                        type     TINYINT NOT NULL DEFAULT 0,
                        creat_time DATETIME DEFAULT CURRENT_TIMESTAMP,
                        UNIQUE KEY uk_user_username (username)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

# 功能设计

## 嵌套查询评论

### ① 一次性查出全部评论

```
List<Comment> allComments = commentMapper.findByPostIdWithUser(postId);
```

对应的 SQL（`CommentMapper.xml`）：

```
SELECT c.id, c.post_id, c.user_id, c.content, c.parent_id, c.create_time,
       u.nickname, u.avatar
FROM t_comment c
LEFT JOIN t_user u ON c.user_id = u.id
WHERE c.post_id = #{postId}
ORDER BY c.create_time DESC
```

这一条 SQL 把该游记下**所有评论（顶级 + 所有层级的回复）**连同用户昵称和头像一次查出来，没有用 N+1 查询。结果通过 `commentWithUserMap` resultMap 映射到 `Comment` 对象，并关联填充 `user` 属性。

------

### ② 构建 `parentId -> 子回复列表` 映射

```
Map<Integer, List<Comment>> childrenMap = allComments.stream()
        .filter(c -> c.getParentId() != null)   // 只取非顶级评论（即有父级）
        .collect(Collectors.groupingBy(Comment::getParentId));
```

用 Java Stream 对所有带 `parentId` 的评论按父 ID 分组，形成一个 **邻接表**：每个父评论 ID 可以快速找到它的直接子回复列表。

------

### ③ 递归组装树形结构

```
List<Comment> topLevel = allComments.stream()
        .filter(c -> c.getParentId() == null)
        .collect(Collectors.toList());

for (Comment comment : topLevel) {
    buildReplies(comment, childrenMap);
}
```

先筛选出顶级评论（`parent_id IS NULL`），然后对每一个顶级评论调用 `buildReplies`：

```
private void buildReplies(Comment parent, Map<Integer, List<Comment>> childrenMap) {
    List<Comment> children = childrenMap.get(parent.getId());
    if (children == null) {
        parent.setReplies(Collections.emptyList());
        return;
    }
    for (Comment child : children) {
        buildReplies(child, childrenMap);
    }
    parent.setReplies(children);
}
```

- 从映射中取出当前评论的直接子回复
- 对每个子回复**递归**调用自身，继续装填更深层的回复
- 将组装好的 `children` 列表设到 `parent.replies` 字段

------

### 数据模型

```
public class Comment {
    private Integer id;
    private Integer postId;
    private Integer userId;
    private String content;
    private Integer parentId;    // null = 顶级评论
    private Date createTime;

    private User user;           // 关联的用户（昵称、头像）
    private List<Comment> replies; // 子回复列表，形成嵌套树
}
```

`replies` 字段是 `List<Comment>` 类型，每个子 `Comment` 又包含自己的 `replies`，从而形成**任意深度的嵌套树结构**。
