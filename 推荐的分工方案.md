## 推荐的分工方案

### 公共部分（两人一起或一人搭好）

```
项目初始化 ├── pom.xml、web.xml、Spring XML 配置 ├── 统一返回类 Result<T>、异常处理 BusinessException ├── 数据库建表 SQL ├── 前端布局模板（header/footer、侧边栏） └── 登录注册（这必须共用一套）
```

### 你负责的模块

| 层面 | 内容                                         |
| ---- | -------------------------------------------- |
| 后端 | **目的地管理** CRUD + **标签管理** CRUD      |
| 后端 | **评论系统**（发表、查看、删除、回复）       |
| 后端 | **文件上传**接口（封面图 + 内容图片）        |
| 前端 | 目的地管理页、标签管理页                     |
| 前端 | 游记详情页的评论区组件                       |
| 前端 | 个人主页（我的游记、我的收藏、个人信息编辑） |

### 搭档负责的模块

| 层面 | 内容                                            |
| ---- | ----------------------------------------------- |
| 后端 | **游记发布/编辑/删除**（核心CRUD + 标签关联）   |
| 后端 | **游记列表查询**（分页 + 多条件搜索 + 动态SQL） |
| 后端 | **收藏切换** + **浏览量统计**                   |
| 前端 | 首页游记列表页（卡片布局、搜索栏、分页）        |
| 前端 | 游记发布/编辑页（富文本编辑器 + 标签选择）      |
| 前端 | 游记详情页主体展示区域                          |

### 流程图（各自独立，不互相等）

```
你：  目的地CRUD ── 标签CRUD ── 评论区 ── 文件上传 ── 个人主页                              ↓ 搭档：游记发布 ── 游记列表(含搜索) ── 游记详情 ── 收藏/浏览量                                    ↓                     联调 + 互相 Review ── 完成
```

------

## 为什么这么分

| 原则           | 说明                                                         |
| -------------- | ------------------------------------------------------------ |
| **CRUD 完整**  | 每人手里都有完整的增删改查模块，评分时看得到工作量           |
| **互不阻塞**   | 你的目的地和标签是搭档的游记依赖的（游记要选目的地和标签），但你优先做，搭档可以先写游记页面用 Mock |
| **代码量均衡** | 两边的 Mapper XML、Service、前端页面数量差不多               |
| **各显技术点** | 你展示文件上传和自关联评论，他展示动态SQL和联表分页查询      |

------

## 合作注意事项

**关键依赖处理**：目的地和标签表你需要**第一周就做完**，搭档写游记发布时才有关联数据可用。在此之前搭档可以先写游记列表页和发布页的 HTML 骨架，数据用 Mock。

**代码冲突**：你们各建各的 Controller 包，比如：

```
controller/ ├── your/ │   ├── DestinationController.java │   ├── TagController.java │   ├── CommentController.java │   └── FileUploadController.java └── partner/    ├── TravelPostController.java    └── FavoriteController.java
```
