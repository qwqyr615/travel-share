# 🏔️ 旅游游记分享平台 — 项目文档

> **课程**：JavaEE 期末大作业
> **技术路线**：传统 Spring + MyBatis + React + Bulma
> **开发方式**：两人全栈分工协作

---

## 1. 项目概述

### 1.1 项目背景

旅游已成为当代人重要的生活方式，但游记分享渠道分散（朋友圈碎片化、专业平台门槛高）。本项目旨在构建一个轻量的**旅游游记分享平台**，用户可以发布图文游记、探索目的地、收藏感兴趣的内容，形成一个围绕旅行的内容社区。

### 1.2 核心目标

- 用户可注册登录、发布/编辑/删除游记
- 游记关联目的地与标签，支持多条件搜索
- 用户可对游记进行评论、收藏
- 管理员可管理目的地和标签
- 响应式设计，适配 PC 和移动端

### 1.3 用户角色

| 角色 | 权限 |
|---|---|
| **游客（未登录）** | 浏览游记列表、查看游记详情、搜索 |
| **注册用户** | 以上全部 + 发布/编辑/删除自己的游记、评论、收藏 |
| **管理员** | 以上全部 + 管理目的地、管理标签、管理所有内容 |

## 4. 数据库设计

### 4.1 E-R 关系图

`
┌──────────┐    1:N    ┌──────────────┐
│   user   │──────────→│  travel_post  │
└──────────┘           └──────────────┘
     │                       │    │
     │ 1:N              N:1  │    │ N:M
     ▼                       ▼    ▼
┌──────────┐          ┌──────────────┐    ┌──────────┐
│ comment  │          │ destination  │    │ post_tag │──N:1──→ tag
└──────────┘          └──────────────┘    └──────────┘
     │
     │ 1:N 自关联（回复）
     ▼
┌──────────┐
│ comment  │ (parent_id)
└──────────┘

┌────────────┐
│  favorite   │──N:1──→ user
│             │──N:1──→ travel_post
└────────────┘
`

### 4.2 各表详细结构

#### 表1：user — 用户表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INT | PRIMARY KEY AUTO_INCREMENT | 用户ID |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 用户名（登录用） |
| password | VARCHAR(100) | NOT NULL | BCrypt 加密存储 |
| nickname | VARCHAR(50) | DEFAULT NULL | 昵称 |
| avatar | VARCHAR(255) | DEFAULT NULL | 头像图片路径 |
| email | VARCHAR(100) | DEFAULT NULL | 邮箱 |
| bio | VARCHAR(300) | DEFAULT NULL | 个人简介 |
| role | VARCHAR(20) | NOT NULL DEFAULT 'user' | 角色：user / admin |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 注册时间 |

#### 表2：destination — 目的地表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INT | PRIMARY KEY AUTO_INCREMENT | 目的地ID |
| name | VARCHAR(100) | NOT NULL | 名称（如"丽江古城"） |
| city | VARCHAR(100) | DEFAULT NULL | 城市 |
| province | VARCHAR(50) | DEFAULT NULL | 省份 |
| cover_image | VARCHAR(255) | DEFAULT NULL | 封面图 |
| description | TEXT | DEFAULT NULL | 简介 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

#### 表3：travel_post — 游记表（核心表）

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INT | PRIMARY KEY AUTO_INCREMENT | 游记ID |
| title | VARCHAR(200) | NOT NULL | 标题 |
| summary | VARCHAR(500) | DEFAULT NULL | 摘要（列表页展示） |
| content | LONGTEXT | NOT NULL | 正文（富文本 HTML） |
| cover_image | VARCHAR(255) | DEFAULT NULL | 封面图路径 |
| user_id | INT | NOT NULL, FK → user(id) | 作者ID |
| destination_id | INT | DEFAULT NULL, FK → destination(id) | 关联目的地 |
| travel_days | INT | DEFAULT NULL | 旅途天数 |
| travel_date | DATE | DEFAULT NULL | 出发日期 |
| budget | DECIMAL(10,2) | DEFAULT NULL | 预算/花费 |
| view_count | INT | DEFAULT 0 | 浏览数 |
| status | VARCHAR(20) | NOT NULL DEFAULT 'published' | 状态：published / draft |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 发布时间 |
| updated_at | DATETIME | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

索引：
`sql
CREATE INDEX idx_post_user ON travel_post(user_id);
CREATE INDEX idx_post_dest ON travel_post(destination_id);
CREATE INDEX idx_post_status_created ON travel_post(status, created_at DESC);
`

#### 表4：tag — 标签表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INT | PRIMARY KEY AUTO_INCREMENT | 标签ID |
| name | VARCHAR(50) | NOT NULL, UNIQUE | 标签名 |

预置数据：
`sql
INSERT INTO tag(name) VALUES ('美食'), ('风景'), ('攻略'), ('民宿'), ('自驾'), ('穷游'), ('摄影');
`

#### 表5：post_tag — 游记-标签关联表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| post_id | INT | NOT NULL, FK → travel_post(id) | 游记ID |
| tag_id | INT | NOT NULL, FK → tag(id) | 标签ID |

联合主键：PRIMARY KEY (post_id, tag_id)

#### 表6：comment — 评论表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INT | PRIMARY KEY AUTO_INCREMENT | 评论ID |
| post_id | INT | NOT NULL, FK → travel_post(id) | 所属游记 |
| user_id | INT | NOT NULL, FK → user(id) | 评论者 |
| parent_id | INT | DEFAULT NULL, FK → comment(id) | 父评论ID（回复用） |
| content | TEXT | NOT NULL | 评论内容 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 评论时间 |

#### 表7：favorite — 收藏表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INT | PRIMARY KEY AUTO_INCREMENT | 收藏ID |
| user_id | INT | NOT NULL, FK → user(id) | 用户ID |
| post_id | INT | NOT NULL, FK → travel_post(id) | 游记ID |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 收藏时间 |

唯一约束：UNIQUE KEY uk_user_post (user_id, post_id)

### 4.3 表间关系总结

| 关系类型 | 示例 | 实现方式 |
|---|---|---|
| 一对多 | user → travel_post | user.id = travel_post.user_id |
| 一对多 | destination → travel_post | destination.id = travel_post.destination_id |
| 一对多 | travel_post → comment | travel_post.id = comment.post_id |
| 多对多 | travel_post ↔ tag | 中间表 post_tag |
| 自关联 | comment → comment（回复） | comment.parent_id |

所有接口统一返回 JSON：

`json

{
  "code": 200,
  "message": "success",
  "data": { ... }
}

// 失败
{
  "code": 400,
  "message": "参数错误",
  "data": null
}

// 未登录
{
  "code": 401,
  "message": "请先登录",
  "data": null
}
`

`json
{
  "code": 200,
  "message": "success",
  "data": {

    "page": 1,

## 7. 部署步骤

### 7.1 环境要求

| 环境 | 版本要求 |
|---|---|
| JDK | 1.8+ |
| Maven | 3.6+ |
| Tomcat | 9.x |
| MySQL | 8.0+ |
| Node.js | 16+（前端构建用） |

### 7.2 数据库初始化

bash:
# 1. 创建数据库
mysql -u root -p
CREATE DATABASE travel_share DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 2. 导入建表脚本
mysql -u root -p travel_share < sql/init.sql

### 7.3 后端部署

bash:
# 1. 修改数据库配置（编辑 src/main/resources/jdbc.properties）
# jdbc.url=jdbc:mysql://localhost:3306/travel_share?useUnicode=true&characterEncoding=utf-8
# jdbc.username=root
# jdbc.password=your_password

# 2. Maven 打包
mvn clean package -DskipTests

# 3. 将 war 包复制到 Tomcat webapps 目录
cp target/travel-share.war /webapps/

# 4. 启动 Tomcat
/bin/startup.sh

# 5. 访问 http://localhost:8080/travel-share/

### 7.4 前端部署

bash:
# 1. 进入前端项目目录
cd frontend

# 2. 安装依赖
npm install

# 3. 配置 API 地址（编辑 src/config.js）
# export const API_BASE_URL = 'http://localhost:8080/travel-share/api';

# 4. 构建生产版本
npm run build

### 7.5 文件上传配置

在 Spring 配置文件中设置 multipart 解析器：

xml:
<bean id="multipartResolver" class="org.springframework.web.multipart.commons.CommonsMultipartResolver">
    <property name="maxUploadSize" value="10485760"/>  <!-- 最大 10MB -->
    <property name="defaultEncoding" value="UTF-8"/>
</bean>

### 7.6 验证启动

bash:
# 1. 查看 Tomcat 日志
tail -f /logs/catalina.out

# 2. 测试健康接口
curl http://localhost:8080/travel-share/api/health

# 3. 预期返回
# {"code":200,"message":"success","data":"TravelShare API is running"}
